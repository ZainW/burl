## Installation Issues

### Binary not found after installation

**Problem**: After installing, `burl` command is not found.

**Solution**: The installation directory may not be in your PATH.

```bash
# Check if directory is in PATH
echo $PATH

# Add to PATH (bash/zsh)
echo 'export PATH="$PATH:$HOME/.burl/bin"' >> ~/.bashrc
source ~/.bashrc

# Add to PATH (fish)
fish_add_path $HOME/.burl/bin
```

### Permission denied when running binary

**Problem**: `Permission denied` error when trying to run burl.

**Solution**: Make the binary executable:

```bash
chmod +x ~/.burl/bin/burl
```

### macOS "cannot be opened because the developer cannot be verified"

**Problem**: macOS blocks unsigned binaries.

**Solution**: Allow the binary to run:

```bash
# Remove quarantine attribute
xattr -d com.apple.quarantine ~/.burl/bin/burl

# Or allow in System Preferences > Security & Privacy
```

## Connection Issues

### Connection timeout

**Problem**: Requests timeout before completing.

**Solutions**:

1. **Increase timeout**:
```bash
burl https://slow-api.com -t 60s
```

2. **Reduce concurrent connections**:
```bash
burl https://api.example.com -c 5
```

3. **Check if server is rate limiting**:
```bash
# Use lower QPS
burl https://api.example.com -q 10
```

### SSL/TLS certificate errors

**Problem**: `SSL certificate problem` or `certificate verify failed`.

**Solutions**:

1. **Skip verification** (development only):
```bash
burl https://self-signed.example.com -k
```

2. **Update system certificates**:
```bash
# macOS
brew install ca-certificates

# Ubuntu/Debian
sudo apt-get update && sudo apt-get install ca-certificates

# Arch Linux
sudo pacman -S ca-certificates
```

### "Failed to resolve host"

**Problem**: DNS resolution fails.

**Solutions**:

1. **Check DNS settings**: Verify your network connection and DNS configuration
2. **Use IP address directly**: `burl http://192.168.1.100:3000`
3. **Check /etc/hosts**: Ensure hostname is not blocked

## Performance Issues

### Unrealistic high RPS numbers

**Problem**: Getting millions of requests per second, which seems wrong.

**Solution**: The target server might be returning cached responses instantly. Try:

1. **Use unique URLs** to bypass cache:
```bash
burl 'https://api.example.com/users?t=$(date +%s)'
```

2. **Use POST requests** (usually not cached):
```bash
burl https://api.example.com/users -m POST -b '{}'
```

### Inconsistent latency results

**Problem**: Latency varies wildly between runs.

**Solutions**:

1. **Add warmup requests**:
```bash
burl https://api.example.com -w 20
```

2. **Increase test duration** for more stable results:
```bash
burl https://api.example.com -d 60s
```

3. **Use latency correction** if coordinated omission is a concern:
```bash
burl https://api.example.com --latency-correction
```

### Low requests per second

**Problem**: Not reaching expected RPS.

**Solutions**:

1. **Increase concurrent connections**:
```bash
burl https://api.example.com -c 100
```

2. **Check if server is rate limiting**:
```bash
# Monitor for 429 status codes
burl https://api.example.com --verbose
```

3. **Use HTTP/2 multiplexing**:
```bash
burl https://api.example.com --http2 -c 50
```

## Output Issues

### TUI not displaying correctly

**Problem**: Rich TUI output is garbled or not showing.

**Solutions**:

1. **Disable TUI**:
```bash
burl https://api.example.com --no-tui
```

2. **Check terminal compatibility**:
```bash
# Verify terminal supports ANSI colors
echo $TERM
# Should show something like "xterm-256color"
```

3. **Update terminal**:
   - iTerm2 (macOS): Recommended
   - Windows Terminal (Windows): Recommended
   - Alacritty, Kitty, WezTerm: All supported

### Colors not showing

**Problem**: Output is plain text without colors.

**Solutions**:

1. **Check if colors are disabled**:
```bash
# Don't use --no-color flag
burl https://api.example.com
```

2. **Verify terminal supports colors**:
```bash
# Test ANSI colors
echo -e "\033[31mRed\033[0m \033[32mGreen\033[0m"
```

3. **Force color output**:
```bash
export FORCE_COLOR=1
burl https://api.example.com
```

### JSON output is invalid

**Problem**: `--llm json` or `-f json` produces invalid JSON.

**Solution**: Ensure you're not mixing output modes:

```bash
# ❌ Wrong - mixes TUI with JSON
burl https://api.example.com --llm json

# ✅ Correct
burl https://api.example.com --llm json --no-tui
```

## Request Issues

### Authentication failing

**Problem**: 401 Unauthorized errors despite providing auth.

**Solutions**:

1. **Check auth format**:
```bash
# Bearer token
burl https://api.example.com -a "bearer:your-token-here"

# Basic auth (user:pass)
burl https://api.example.com -a "basic:username:password"
```

2. **Use header instead**:
```bash
burl https://api.example.com \
  -H "Authorization: Bearer your-token"
```

3. **Verify token is valid**:
```bash
# Test with curl first
curl -H "Authorization: Bearer $TOKEN" https://api.example.com
```

### Request body not being sent

**Problem**: Server reports missing request body.

**Solutions**:

1. **Ensure Content-Type is set**:
```bash
burl https://api.example.com \
  -m POST \
  -b '{"key":"value"}' \
  -T application/json
```

2. **Use body file for complex payloads**:
```bash
burl https://api.example.com \
  -m POST \
  -B ./payload.json \
  -T application/json
```

### Custom headers not working

**Problem**: Custom headers don't seem to be sent.

**Solution**: Check header format (note the colon and space):

```bash
# ❌ Wrong
burl https://api.example.com -H "X-Custom-Header=value"

# ✅ Correct
burl https://api.example.com -H "X-Custom-Header: value"
```

## Platform-Specific Issues

### Linux: "error while loading shared libraries"

**Problem**: Missing system libraries.

**Solution**:

```bash
# Ubuntu/Debian
sudo apt-get install libc6 libgcc1 libstdc++6

# Arch
sudo pacman -S glibc gcc-libs
```

### Windows: "Access is denied"

**Problem**: Windows Defender or antivirus blocking execution.

**Solutions**:

1. **Add exception in Windows Defender**:
   - Windows Security > Virus & threat protection
   - Manage settings > Exclusions > Add exclusion
   - Add the burl.exe file

2. **Run from trusted location**:
   - Move binary to `C:\Program Files\burl\`
   - Add to PATH

### macOS: Performance degradation

**Problem**: burl runs slower on macOS than expected.

**Solutions**:

1. **Use native ARM binary on Apple Silicon**:
```bash
# Download darwin-arm64, not darwin-x64
curl -L https://github.com/ZainW/burl/releases/latest/download/burl-darwin-arm64 -o burl
```

2. **Check Activity Monitor** for other resource-intensive processes

## Getting Help

If you're still experiencing issues:

1. **Check diagnostic mode**:
```bash
burl https://api.example.com --diagnose
```

2. **Run with verbose output**:
```bash
burl https://api.example.com -v
```

3. **Open an issue** on GitHub with:
   - burl version (`burl --version`)
   - Operating system and version
   - Command you're running
   - Error message or unexpected behavior
   - Diagnostic output if relevant

**GitHub Issues**: [github.com/ZainW/burl/issues](https://github.com/ZainW/burl/issues)
